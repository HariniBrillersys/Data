# PowerPoint MCP Server

MCP server for generating PowerPoint presentations from corporate templates. Provide a `.potx` or `.pptx` template -- the server auto-detects layouts, fills placeholders, and preserves the template's branding and design.

## How It Works

1. Provide a PowerPoint template (`.potx` or `.pptx`)
2. The server auto-detects all layouts and placeholder roles
3. AI agents provide structured content; the server selects layouts and fills placeholders
4. Output is a `.pptx` with correct fonts, colors, and positioning from the template

## Quick Start

### 1. Install

```bash
pip install git+https://github.com/merckgroup/powerpoint-mcp.git
```

### 2. Add your template

```bash
mkdir ~/pptx-templates
cp /path/to/YourCompany.potx ~/pptx-templates/
```

### 3. Run

```bash
PPTX_TEMPLATES_DIR=~/pptx-templates pptx-mcp
```

## Installation Options

The base install includes chart generation (matplotlib) and all presentation features. AI image generation requires the `[images]` extra.

| Method | Command |
|--------|---------|
| **pip** | `pip install git+https://github.com/merckgroup/powerpoint-mcp.git` |
| **uv** | `uv pip install git+https://github.com/merckgroup/powerpoint-mcp.git` |
| **pip + AI images** | `pip install "pptx-mcp[images] @ git+https://github.com/merckgroup/powerpoint-mcp.git"` |
| **uv + AI images** | `uv pip install "pptx-mcp[images] @ git+https://github.com/merckgroup/powerpoint-mcp.git"` |
| **Dev** | `git clone ... && pip install -e ".[dev]"` |

The `[images]` extra adds `openai` for AI image generation via the UPTIMIZE API. The `[dev]` extra includes `[images]` plus testing and linting tools.

After installing, `pptx-mcp` is available globally. You can also run `python -m pptx_mcp`.

## Connecting to AI Clients

### OpenCode

Add to your `opencode.json`:

```json
{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "powerpoint": {
      "type": "local",
      "command": ["pptx-mcp"],
      "enabled": true,
      "environment": {
        "PPTX_TEMPLATES_DIR": "/path/to/your/templates",
        "UPTIMIZE_OPENAI_API_KEY": "your-api-key-here",
        "UPTIMIZE_ENV": "dev"
      }
    }
  }
}
```

### Claude Desktop

Add to `~/Library/Application Support/Claude/claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "powerpoint": {
      "command": "pptx-mcp",
      "env": {

        "UPTIMIZE_OPENAI_API_KEY": "your-api-key-here",

```

### Remote (SSE)

```bash
PPTX_TRANSPORT=sse PPTX_PORT=8000 PPTX_TEMPLATES_DIR=/shared/templates pptx-mcp
```

## Configuration

All configuration via environment variables:

| Variable | Default | Description |
|----------|---------|-------------|
| `PPTX_TEMPLATES_DIR` | `./templates` | Folder containing template files |
| `PPTX_OUTPUTS_DIR` | `./outputs` | Where generated presentations are saved |
| `PPTX_DEFAULT_TEMPLATE` | *(empty)* | Default template name |
| `PPTX_TRANSPORT` | `stdio` | Transport: `stdio` or `sse` |
| `PPTX_HOST` | `127.0.0.1` | SSE bind host |
| `PPTX_PORT` | `8000` | SSE port |
| `PPTX_LOG_LEVEL` | `INFO` | Logging level: DEBUG, INFO, WARNING |
| `UPTIMIZE_OPENAI_API_KEY` | *(empty)* | API key for UPTIMIZE OpenAI image generation |
| `UPTIMIZE_ENV` | `dev` | UPTIMIZE environment: `dev` or `p` |

## Slide Content Schema

Each slide in the `slides` list is a dict:

| Key | Type | Used For |
|-----|------|----------|
| `layout` | str | Preferred layout name (auto-selected if omitted) |
| `title` | str | Slide title |
| `subtitle` | str | Subtitle (title slides) |
| `body` | str | Sub-headline / category text |
| `content` | str or list | Main content area (bullets or paragraph) |
| `content_left` | str or list | Left column (two-column layouts) |
| `content_right` | str or list | Right column (two-column layouts) |
| `content_1/2/3` | str or list | Numbered columns (three-column layouts) |
| `notice` | str or list | Sidebar panel |
| `image` | str or list | Path to image, or list of paths/dicts for multi-image (max 9) |
| `image_mode` | str | `"fill"` (crop), `"fit"` (no crop, grid), or `"collage"` (overlapping) |
| `notes` | str or list | Speaker notes for Presenter View (keep brief) |
| `shapes` | list[dict] | Decorative shape annotations (see below) |

## Shape Annotations

Slides can include decorative shapes layered on top of template content. Shapes use the template's theme colors for brand consistency.

```python
{
    "title": "Revenue Growth",
    "content": ["Q1: $10M", "Q2: $15M"],
    "shapes": [
        {"type": "arrow", "direction": "right", "label": "+50%", "color": "accent2"},
        {"type": "badge", "text": "Record Quarter", "position": "top-right", "color": "accent1"},
    ]
}
```

### Shape Types

| Type | Description | Key Params |
|------|-------------|------------|
| `arrow` | Directional arrow | `direction` (right/left/up/down), `label` |
| `callout` | Text callout box | `text` |
| `badge` | Small label tag | `text` |
| `highlight` | Semi-transparent overlay | `opacity`, `width`, `height` |
| `connector` | Line between points | `start`, `end`, `style` (solid/dashed) |
| `process_arrow` | Chevron process flow | `steps` (list of labels) |

### Positioning

- Named positions: `top-left`, `top-right`, `center`, `bottom-left`, `below-title`, etc.
- `auto`: Intelligent default placement
- Custom: `{"left": emu, "top": emu}`

### Colors

- Theme colors: `accent1` through `accent6`, `dark`, `light`
- Direct hex: `#0F69AF`
- Custom names from template: `Rich Purple`, `Vibrant Cyan`, etc.

## Chart Generation

Generate charts as PNG images that can be placed into slide picture placeholders.

```python
# Generate chart
result = generate_chart(
    chart_type="bar",
    data={"Q1": 10, "Q2": 15, "Q3": 22, "Q4": 28},
    title="Revenue by Quarter ($M)",
    ylabel="Revenue ($M)",
)

# Use in a slide (fit mode preserves the full chart without cropping)
create_presentation(slides=[
    {"title": "Revenue", "image": result["path"], "image_mode": "fit"}
])
```

### Supported Chart Types

`bar`, `horizontal_bar`, `stacked_bar`, `line`, `pie`, `donut`, `scatter`

### Multi-Series Data

```python
generate_chart(
    chart_type="line",
    data=[
        {"name": "Revenue", "values": {"Q1": 10, "Q2": 15, "Q3": 22}},
        {"name": "Costs", "values": {"Q1": 8, "Q2": 11, "Q3": 14}},
    ],
)
```

Charts are a core feature and always available. They automatically use the template's color scheme.
